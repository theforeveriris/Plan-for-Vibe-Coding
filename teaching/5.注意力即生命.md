# 注意力即生命

在 AI 辅助编程中，注意力管理是提高效率的关键。本文档详细介绍如何有效管理 AI 的注意力，避免上下文丢失和信息过载。

## 1. 会话分割策略（最重要）

每 30-45 分钟主动重置会话，不要等到 AI 开始失忆才行动。

**具体操作：**

```
1. 当前会话结束前：「请用一段话总结我们目前的进展、关键决策、
   未完成事项和下一步目标，要足够详细以便在新会话中继续」

2. 开启新会话，粘贴总结 + 具体的新任务
```

这比压缩更可靠，因为压缩是有损的，而手写的交接笔记是你筛选过的重要信息。

## 2. 建立 CLAUDE.md / 项目记忆文件

这是跨会话持久化的关键。Claude Code 会自动读取项目根目录的 CLAUDE.md 文件。

**应该包含：**

- 技术栈约束（"使用 ES 模块而非 CommonJS"）
- 命名规范、文件组织方式
- 常见错误的规避规则（每次纠正 AI 后，让它更新此文件）

**红线：** 超过 500 行会导致 AI 注意力分散，只保留"没有这条规则 AI 就会犯错"的内容。

## 3. 任务分块 + Plan Mode

不要在一个会话里完成"调研→设计→编码→测试"全流程。

**正确姿势：**

- **Plan Mode**（Claude Code 中按 Shift+Tab）：只思考不编码，产出技术方案
- 新会话：基于方案执行具体编码
- 再开新会话：代码审查或测试

每个阶段结束后 /compact 或重启会话，保持上下文干净。

## 4. 子代理（Subagents）策略

把"脏活"扔给子代理，保护主会话的上下文。

**示例：**

```
主会话：「使用子代理调查我们的认证系统如何处理 token 刷新，
 并检查是否有可复用的 OAuth 工具」

子代理：读取 20 个文件、分析日志 → 返回 500 字摘要

主会话：基于摘要做决策，上下文几乎零损耗
```

## 5. 并行会话 + Git Worktree

对于复杂功能，同时开多个终端：

- 会话 A：写功能代码
- 会话 B：审查 A 的代码（基于 git worktree 的独立工作区）
- 会话 C：写测试用例

通过 git worktree 创建独立的代码副本，互不干扰。

## 6. 关于 Memory MCP 的替代方案

如果感觉 Memory MCP 力不从心，可以尝试：

- **Sequential Thinking MCP**：专门用于维护思维链，适合复杂推理
- **Context Window Management Skill**：Claude Code 的技能系统，提供分层的上下文路由和智能摘要

或者完全放弃 MCP 记忆，改用文件系统：

- 每个功能一个 task-xxx.md 文档
- 包含：背景、决策记录、代码片段、待办
- 新会话开始时让 AI 读取相关文档
