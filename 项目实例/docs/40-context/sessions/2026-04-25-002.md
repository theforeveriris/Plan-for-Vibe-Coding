# Session 2026-04-25-002

## 任务
Dashboard 布局重构与 Bug 修复：合并控制面板/实时终端/已发现密钥到单个 TabContainer，修复布局挤压和滚动问题

## 完成的工作

### 1. 合并三个组件到单个 TabContainer
- **文件**: `src/frontend/src/views/Dashboard.vue`
- **变更**:
  - 将原来的 StatsPanel + MinerControl + TabContainer(终端/密钥) 改为 StatsPanel + TabContainer(控制/终端/密钥)
  - 三个标签：`控制面板`、`实时终端`、`已发现的特殊密钥`
  - 默认激活 `控制面板` 标签
  - 左侧面板现在只有两部分：StatsPanel（固定高度）+ TabContainer（占据剩余空间）

### 2. 修复 TabContainer 被挤压导致不可见
- **文件**: `src/frontend/src/views/Dashboard.vue`
- **原因**: `left-panel` 是 flex 纵向布局，StatsPanel 和 MinerControl 没有限制高度，会撑开内容，导致 TabContainer 被挤压到几乎看不见
- **修复**:
  - StatsPanel（第1个子元素）：`flex-shrink: 0` - 保持自然高度
  - TabContainer（第2个子元素）：`flex: 1; min-height: 0; overflow: hidden` - 占据剩余所有空间

### 3. 修复 MinerControl 无法滚动
- **文件**: `src/frontend/src/components/MinerControl.vue`
- **原因**: MinerControl 组件没有设置 `overflow-y: auto`，当规则数量多、内容超出容器高度时无法滚动
- **修复**: 添加 `overflow-y: auto` 到 `.miner-control` 样式

### 4. 修复前端服务未启动问题
- 启动前端开发服务器 `pnpm dev` 在 http://localhost:5173/

## 关键决策
- 使用 `v-show` 而非 `v-if` 切换标签内容，避免组件重复挂载/卸载导致的性能问题和状态丢失
- TabContainer 内的 `.tab-panel` 设置 `overflow-y: auto; overflow-x: hidden`，确保每个标签内容独立可滚动
