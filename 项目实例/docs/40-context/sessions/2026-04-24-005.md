# Session 2026-04-24-005

## 任务
修复 SSE 连接持续断开并最终重连失败的问题

## 问题分析

日志显示任务ID为 40078e55-4ea0-4cff-b67d-c2d17aab39c2 和 c15f7963-db2c-4ea8-b305-648aaad591dc 的两次任务均出现相同的连接问题：SSE连接在建立后立即断开，经过5次指数退避重连（2秒、4秒、8秒、16秒、30秒间隔）后达到上限并失败。

### 根本原因

1. **EventSource `onerror` 触发时机问题**：在连接建立但无数据时也会触发 `onerror`
2. **keep-alive 间隔过长**：30秒间隔可能导致某些代理/防火墙断开空闲连接
3. **前端 `onmessage` 未处理 `event.data` 为空的情况**
4. **缺少连接健康检测**：无法区分"连接正常但无数据"和"连接已断开"
5. **重连策略过于激进**：指数退避从2秒开始，5次后达到上限，没有网络状态检测
