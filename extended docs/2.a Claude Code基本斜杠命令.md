# Claude Code 基本斜杠命令

本教程详细介绍 Claude Code 的斜杠命令、CLI 命令、快捷键等功能，帮助开发者高效使用 Claude Code。

## 一、斜杠命令

以 / 开头的命令用于快速执行特定操作：

| 命令 | 功能 | 使用场景 |
|------|------|---------|
| /clear | 清空对话上下文 | 开始新任务时 |
| /model | 切换 AI 模型 | 需要更强能力时切换到 Opus |
| /status | 查看使用额度和计费 | 检查剩余额度 |
| /config | 打开配置界面 | 修改设置 |
| /resume | 恢复最近的会话 | 重启后继续之前的工作 |
| /rewind | 恢复到上一个检查点 | 代码改错需要回退 |
| /agents | 管理 Agent | 创建/查看自定义 Agent |
| /init | 生成 CLAUDE.md 模板 | 新项目快速配置 |
| /compact | 压缩对话上下文 | 上下文太多时精简 |
| /export | 导出对话记录 | 分享或保存对话 |
| /statusline | 自定义状态栏显示 | 隐藏/显示状态信息 |
| /vim | 启用 Vim 键绑定 | 熟悉 Vim 的用户 |
| /fork | 分叉当前会话 | 尝试不同方案而不丢失原线程 |
| /mcp | 管理 MCP 服务器 | 启用/禁用 MCP 集成 |

常用场景示例：

```bash
# 开新任务时清空上下文
/clear

# 查看剩余额度
/status

# 切换到更强模型
/model opus

# 恢复到之前的状态
/rewind
```

## 二、CLI 命令与启动选项

| 命令 | 描述 | 示例 |
|------|------|------|
| claude | 启动交互式 REPL | claude |
| claude "query" | 使用初始提示启动 REPL | claude "解释这个项目" |
| claude -p "query" | 查询后退出（无头模式） | claude -p "检查代码类型错误" |
| claude -c | 继续上一次对话 | claude -c |
| claude -r "id" | 恢复指定会话 | claude -r "abc123" |
| claude --continue | 加载最近的对话 | claude --continue |
| claude --resume | 显示会话选择器 | claude --resume |

启动选项：

| 选项 | 功能 | 示例 |
|------|------|------|
| -p "query" | 执行查询后退出 | claude -p "运行测试" |
| --model | 指定模型 | claude --model opus |
| --permission-mode | 设置权限模式 | claude --permission-mode plan |
| --add-dir | 添加工作目录 | claude --add-dir ../shared |
| --fork-session | 分叉会话 | claude -c --fork-session |
| --dangerously-skip-permissions | 跳过权限提示（危险） | claude --dangerously-skip-permissions |
| --allowedTools | 允许的工具列表 | claude --allowedTools "Read,Bash" |
| --disallowedTools | 禁止的工具列表 | claude --disallowedTools "Edit" |
| --system-prompt | 替换整个系统提示 | claude --system-prompt "..." |
| --system-prompt-file | 从文件加载系统提示 | claude -p --system-prompt-file ./prompt.txt |
| --append-system-prompt | 附加到默认提示 | claude --append-system-prompt "..." |
| --output-format | 输出格式（text/json/stream-json） | claude -p --output-format json |
| --verbose | 启用详细日志 | claude --verbose |
| --max-turns | 限制轮数 | claude -p --max-turns 3 |

系统提示标志区别：

| 标志 | 行为 | 模式 | 用例 |
|------|------|------|------|
| --system-prompt | 替换整个默认提示 | 交互 + 打印 | 完全控制行为 |
| --system-prompt-file | 替换为文件内容 | 仅打印 | 从文件加载 |
| --append-system-prompt | 附加到默认提示 | 交互 + 打印 | 添加特定指令 |

## 三、快捷键

| 快捷键 | 功能 | 上下文 |
|-------|------|------|
| Ctrl+C | 取消当前输入或生成 | 标准中断 |
| Ctrl+D | 退出会话 | EOF 信号 |
| Ctrl+L | 清除终端屏幕 | 保留对话历史 |
| Ctrl+R | 反向搜索命令历史 | 搜索以前的命令 |
| Esc+Esc | 回退代码/对话 | 恢复到之前状态 |
| Tab | 切换扩展思考 | 开启/关闭思考模式 |
| Shift+Tab | 切换权限模式 | 循环切换权限模式（Normal/Auto-accept/Plan） |
| Ctrl+B | 后台运行命令 | 将 Bash 调用移到后台 |
| Ctrl+_ | 撤销最后操作 | 撤销上一次编辑 |

多行输入方法：

| 快捷键 | 上下文 |
|-------|------|
| \ + Enter | 适用于所有终端 |
| Option+Enter (macOS) | macOS 默认设置 |
| Shift+Enter | 配置后可用 |

| 前缀 | 功能 | 示例 |

## 五、Vim 模式

使用 /vim 启用或通过 /config 永久配置。

模式切换：

| 命令 | 操作 | 来自模式 |
|------|------|--------|
| Esc | 进入 NORMAL 模式 | INSERT |
| i | 在光标前插入 | NORMAL |
| a | 在光标后插入 | NORMAL |
| o | 在下方打开行 | NORMAL |

导航（NORMAL 模式）：

| 命令 | 操作 |
|------|------|
| h/j/k/l | 左/下/上/右移动 |
| w | 下一个单词 |
| b | 上一个单词 |
| 0/$ | 行首/行尾 |
| gg/G | 输入开始/结束 |

## 六、后台运行

工作原理：
- 异步运行命令，立即返回任务 ID
- 输出被缓冲，可用 BashOutput 工具检索
- Claude Code 退出时自动清理

常见后台命令：
- 构建工具（webpack, vite, make）
- 包管理器（npm, yarn, pnpm）
- 测试运行器（jest, pytest）
- 开发服务器

按 Ctrl+B 将常规 Bash 调用移到后台。

## 七、Bash 模式（! 前缀）

```bash
! npm test
! git status
! ls -la
```

- 将命令和输出添加到对话上下文
- 显示实时进度
- 支持 Ctrl+B 后台运行

## 八、Agents（智能体）

创建 Agent：
- 使用 /agents 创建新 Agent，或在以下路径手动创建 Markdown 文件：
  - .claude/agents/code-reviewer.md（项目级）
  - ~/.claude/agents/code-reviewer.md（全局）

必需字段：
- description：何时调用（自然语言）
- prompt：系统提示

可选字段：
- tools：可用工具数组
- model：模型别名（sonnet/opus/haiku）

使用 --agents 标志启动：

```bash
claude --agents '{
  "code-reviewer": {
    "description": "Expert code reviewer",
    "prompt": "You are a senior code reviewer",
    "tools": ["Read", "Grep", "Glob", "Bash"],
    "model": "sonnet"
  }
}'
```

提示词技巧：
- 使用 think、think deeply、think hard、ultrathink 等关键词可触发不同层级的深度思考模式

## 十、Git 集成

Claude Code 具有深度的 Git 集成和内置安全协议。

安全协议：
- 自动尊重 .gitignore
- 不会未经许可修改分支
- 提交前显示差异
- 遵循项目的提交规范
- 未经明确请求从不强制推送
- 修改提交前检查作者身份

常用工作流：

```bash
# 创建提交
> commit these changes
> create a commit with a meaningful message

# 创建 PR
> create a PR for this feature
> summarize the changes and create a PR

# 代码审查
> review the changes in this PR
> what could go wrong with these changes?
```

禁用提交/PR 署名：

默认情况下，Claude Code 会添加 Co-Authored-By 署名。可在 ~/.claude/settings.json 中禁用：

```json
{
  "attribution": {
    "commit": "",
    "pr": ""
  }
}
```

## 十一、最佳实践与技巧

上下文管理：AI 上下文像牛奶一样，越新鲜越好。定期使用 /compact 压缩上下文，或在无关任务之间使用 /clear。

模型选择策略：
- Sonnet：默认用于约 80% 的任务
- Haiku：重命名、样板代码、格式化、简单查找
- Opus：架构决策、复杂调试、细致的代码审查

智能文件读取：使用 grep 和 find 定位内容，避免一次性读取整个文件。

禁用未使用的 MCP 服务器：每个 MCP 服务器消耗约 5-15K tokens 的上下文，使用 /mcp 管理。

容器化运行：对于长时间运行或有风险的任务，在 Docker 容器中运行 Claude Code，使用 --dangerously-skip-permissions 实现完全自主的"工作器"。

子代理策略：将大任务分解为多个子代理并行处理，可以指定数量、前后台运行、选择模型。

指数退避检查：对于长时间运行的任务（如 Docker 构建、GitHub CI），可以让 Claude Code 使用指数退避（1分钟、2分钟、4分钟...）手动检查状态。

## 十二、GitHub Actions 集成

```yaml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]
  issue_comment:
    types: [created]

jobs:
  claude-review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Review this PR for bugs, security issues, and performance problems"
          claude_args: "--max-turns 5"
```

## 十三、CLAUDE.md 与 Skills

全局配置（~/.claude/CLAUDE.md）：适用于所有项目的通用规则。

项目级配置（.claude/CLAUDE.md）：特定项目的上下文和规则。

Skills：可复用的工作流模板，存储在 .claude/skills/ 目录下，懒加载机制，仅在需要时注入上下文。
